## 데모 한 곡의 완료 조건

한 곡의 데모는 `세션`과 `마감` 두 국면으로 쪼개서 본다. 한 번에 다 끝내지 못해도 괜찮다. 깬 항목만큼 진전으로 인정하고, 남은 항목은 다음 세션에서 이어서 닫는다.

### 세션 (60분, 악기 앞에서)

- [ ] 튜닝 + 현재 키 확인
- [ ] 후렴·최고음으로 키 테스트
- [ ] 메트로놈 BPM 후보 2개 비교
- [ ] 멈추지 않은 전체 1테이크 확보
- [ ] 바로 청취하며 불편한 구간 표시

### 마감 (세션 직후, 책상에서)

- [ ] 키·카포·BPM·구조 곡 문서에 기록
- [ ] 얹어볼 악기·질감 1개 메모
- [ ] 다음 편곡 아이디어 정리
- [ ] 파일명 정리 + 곡 문서·세션 노트 연결
- [ ] 다음에 이어갈 것 1줄 기록

## 두 국면을 나눠서 보는 기준

### 세션 — 악기 앞에서만 할 수 있는 것

- 처음부터 끝까지 끊기지 않은 통기타+보컬 파일 1개. 필요하면 후렴·브릿지처럼 애매한 구간만 짧은 추가 테이크.
- 키와 카포 위치, BPM 후보 또는 자유 템포 여부를 그 자리에서 손으로 확인한다.
- 전체 테이크를 바로 들으며 편한 구간과 불편한 구간을 표시한다.

### 마감 — 책상에서 정리하는 것

- 키·카포·BPM·현재 구조와 길이를 곡 문서에 옮긴다.
- 패드, 퍼커션, 코러스 기타, 베이스 톤처럼 얹어볼 요소 1개를 메모한다. 실제 오버더빙을 하거나 말로만 적어도 인정하며, 중요한 것은 “다음 편곡 실험 출발점”을 남기는 것이다.
- 방금 남긴 메모를 바탕으로 다음 편곡에서 바로 시험할 아이디어를 한 줄로 정리한다. 예를 들면 “패드 먼저”, “후렴 스트로크 더 세게”, “2절 베이스 후보 듣기”처럼 다음 행동이 보이면 충분하다.
- 파일 경로를 곡 문서·세션 노트에 연결하고, 다음에 이어갈 것을 한 줄로 남긴다.

연주 실수, 음정 실수와 생활 소음은 키/BPM 판단을 방해하지 않는 한 다시 녹음할 이유가 아니다.

## 시간이 부족한 날의 최소 데모

30분밖에 없다면 다음만 수행한다.

1. 후렴 최고음으로 키를 확인한다.
2. 임시 BPM 하나를 선택한다.
3. 멈추지 않고 전체를 한 번 녹음한다.
4. 파일명에 날짜와 버전을 적고, 얹어볼 악기나 다음 아이디어를 한 줄 남긴다.

이 최소 데모도 판단이 가능하면 완료로 인정한다.
