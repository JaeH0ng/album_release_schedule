# 싱어송라이터 공모전 운영

이 문서는 앨범 제작 일정과 별도로, 싱어송라이터 성격의 공모전 후보를 어떻게 추적하고 웹에 반영할지 정리한다.

## 목적

- 매주 싱어송라이터 공모전을 다시 찾는다.
- 실제로 음악가 개인 프로젝트에 도움이 되는 공고만 남긴다.
- 웹에서 `수락`, `보류`, `제외`를 빠르게 판단한다.
- `수락`한 공모전은 캘린더에 넣어 마감 리마인드로 쓴다.

## 데이터 기준

- 공모전 후보는 `public.singer_songwriter_opportunities` 테이블을 기준으로 본다.
- 웹의 카드 목록은 `Supabase`에서 최신 공모전 후보를 읽는다.
- 웹에서 누르는 `수락`, `보류`, `제외`는 로그인한 사용자의 `public.opportunity_reviews`에 저장된다.
- 로그인 전에는 현재 브라우저의 임시 저장 상태를 사용한다.
- `수락`한 공모전은 웹 캘린더에 마감 일정으로 합쳐진다.

## 동기화 방식

- 로그인은 `Supabase Auth` 이메일 매직링크를 사용한다.
- 같은 이메일로 폰과 PC에서 로그인하면 공모전 판단 상태가 같은 계정으로 동기화된다.
- 웹 공개 일정은 누구나 읽을 수 있지만, 공모전 판단 쓰기는 로그인한 사용자 본인만 가능하다.
- 관리자 편집은 `public.admin_users`에 등록된 이메일만 가능하다.

## 관리자 모드

- 관리자 계정은 `public.admin_users` 화이트리스트로 판별한다.
- 관리자에게는 웹의 `공모전 관리자 패널`이 열리고, 여기서 공모전 추가/수정/삭제가 가능하다.
- 일반 로그인 사용자는 공모전 판단 동기화만 가능하고, 공고 원본 데이터는 수정할 수 없다.

## 상태 규칙

- `open`: 현재 접수 중이거나 마감일이 확인된 공고
- `watch`: 싱어송라이터와 잘 맞지만 아직 이번 회차 공고를 더 확인해야 하는 항목
- `closed`: 올해 모집은 종료되었지만 다음 시즌 추적 가치가 있는 항목

## 매주 월요일 업데이트 루틴

1. 공식 사이트와 공식 공고를 우선 확인한다.
2. 싱어송라이터, 자작곡, 신인 창작 뮤지션 중심인지 먼저 본다.
3. 접수 기간, 마감일, 제출 형식, 주최를 테이블에 반영한다.
4. 이미 끝난 공고는 `closed`로 바꾸고, 다시 열리면 `open`으로 복구한다.
5. 웹에서 바로 판단할 수 있게 준비물 한 줄을 남긴다.
