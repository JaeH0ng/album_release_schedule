# 프로젝트 컨텍스트

이 문서는 새로운 Codex 세션에서도 지금까지 합의한 앨범 제작 방향과 일정을 이어가기 위한 기준 정보다.

## 아티스트와 앨범

- 아티스트가 작곡, 작사와 프로듀싱을 직접 담당한다.
- 정규 앨범이며 현재 후보곡은 10곡, 최종 수록 목표는 8~9곡이다.
- 작사와 작곡은 완료되었고, 가사 정제도 완료되었다.
- 중심 악기는 통기타와 보컬이다.
- 통기타+보컬 데모는 완성 녹음이 아니라 키, BPM, 박자와 구조를 찾기 위한 출발점이다.
- 여러 악기로 편곡 테스트를 마친 뒤 통기타와 보컬을 다시 녹음하는 것을 본녹음으로 정의한다.

## 현재 후보곡

현재 번호는 관리용 임시 번호이며 최종 트랙 순서가 아니다.

1. Psyche
2. 괜한 말
3. 날 좀 봐줘요, 좀 봐줘요
4. 누군가의
5. 대동제
6. 또다시
7. 새벽 두 시
8. 소란스러운 밤
9. 스물 여덟
10. good night

정제 가사의 기준본은 `lyrics` 폴더에 있고, 곡별 제작 정보는 `tracks` 폴더에 있다. 가사 표현과 맞춤법은 아티스트의 명시적 요청 없이 임의로 수정하지 않는다.

## 데모 완료 기준

- 처음부터 끝까지 이어지는 통기타+보컬 파일 1개
- 녹음한 키와 카포 위치
- 임시 BPM 또는 자유 템포 여부
- 현재 곡 구조
- 가장 편한 구간과 가장 불편한 구간
- 다음에 테스트할 한 가지

곡당 기본 60분, 최대 90분으로 제한한다. 실수가 있어도 키, BPM과 구조를 판단할 수 있으면 완료로 인정한다.

## 제작 운영 원칙

- 작업 시간이 부족한 것이 가장 큰 병목이다.
- 작업 장소와 녹음실 선택은 아티스트가 별도로 결정한다.
- 매주 녹음 시간을 먼저 고정하고, 메인 블록이 무너지면 30분 최소 세션으로 결과물을 남긴다.
- 한 곡이 늦어져도 다음 곡의 마감을 함께 미루지 않는다.
- 같은 데모를 세 번 이상 다시 녹음하지 않는다.
- 편곡이 확정된 곡은 다른 곡을 기다리지 않고 본녹음으로 넘긴다.
- 믹스와 유통 검수 기간을 줄이지 않는다. 일정이 밀리면 수록 범위나 선택적 파트를 줄인다.

## Codex와 작업하는 방식

매주 다음 정보를 바탕으로 현실적인 목표를 다시 계산한다.

```text
이번 주 가능한 시간:
완료한 작업:
꼭 해야 하는 작업:
막힌 부분:
```

녹음 전에는 곡, 확보 시간과 현재 상태를 전달해 세션 계획을 만들고, 녹음 후에는 파일 경로, 결과와 다음 행동을 곡 문서에 기록한다.

일정이나 곡 상태를 변경하면 다음 파일을 함께 갱신한다.

- `SCHEDULE.md`
- `DEMO_PLAN.md`
- `TRACK_STATUS.md`
- 해당 `tracks/*.md`
- 웹 캘린더 데이터가 바뀌면 `app.js`
